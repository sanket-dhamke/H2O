# Deploying H2O (backend + database) to the cloud

This makes the backend reachable from anywhere (phones, store builds), instead
of only your laptop. We use **Supabase** (free Postgres) + **Render** (free web
service).

The mobile/web app talks to this hosted backend by URL, so once this is live you
can build installable apps.

---

## Step 1 — Create the database (Supabase)

1. Go to https://supabase.com and sign up (free).
2. **New project** → pick a name, set a strong **database password** (save it),
   choose the region closest to you.
3. Wait ~2 min for it to provision.
4. Click **Connect** (top bar) → **ORMs** (or **Connection string**) → choose
   **Prisma** / **Session pooler**. Copy the connection string. It looks like:
   ```
   postgresql://postgres.abcdefgh:YOUR-PASSWORD@aws-0-xxx.pooler.supabase.com:5432/postgres
   ```
   - Replace `YOUR-PASSWORD` with the password from step 2.
   - If the password has special characters, URL-encode them (e.g. `@` → `%40`).

➡️ **Send me this connection string** and I'll create + seed the schema on it.

---

## Step 2 — Put the code on GitHub (Render deploys from Git)

From the project root (`C:\Users\sdhamke\h2o`):

```bash
git init
git add .
git commit -m "H2O society app"
```

Then create an **empty** repo on https://github.com (e.g. `h2o`), and:

```bash
git remote add origin https://github.com/<you>/h2o.git
git branch -M main
git push -u origin main
```

`.env` files are git-ignored, so your secrets are NOT pushed.

---

## Step 3 — Deploy the backend (Render)

1. Go to https://render.com and sign up (free); connect your GitHub.
2. **New +** → **Blueprint** → pick your `h2o` repo. Render reads `render.yaml`
   and creates the `h2o-api` web service.
3. In the service's **Environment**, set the secrets it asks for:
   - `DATABASE_URL` = the Supabase string from Step 1
   - `RAZORPAY_KEY_ID`, `RAZORPAY_KEY_SECRET` = your keys
   - `AI_BASE_URL=https://api.groq.com/openai/v1`, `AI_API_KEY=<groq key>`,
     `AI_CHAT_MODEL=llama-3.3-70b-versatile`, `AI_TRANSCRIBE_MODEL=whisper-large-v3`
   - (`JWT_SECRET` is auto-generated by the blueprint.)
4. Deploy. When it's live you'll get a URL like `https://h2o-api.onrender.com`.
5. Verify: open `https://h2o-api.onrender.com/api/health` → should show
   `{"ok":true,...}`.

> Note: the free plan sleeps after ~15 min idle, so the first request after
> idle takes a few seconds to wake. Fine for testing.

---

## Step 4 — Point the app at the hosted backend

Give me the Render URL and I'll set `app.json` → `extra.apiUrl` to it (and the
web client will use it automatically). After that we can build the Android app.

## One-time database seed

The build runs `prisma db push` (creates tables). To load the demo
flats/users/bills once, run this in Render's **Shell** tab (or locally with the
Supabase `DATABASE_URL`):

```bash
node src/seed.js
```

Demo logins after seeding: `admin@h2o.com`, `guard@h2o.com`, `resident@h2o.com`
(password `Password123`).
